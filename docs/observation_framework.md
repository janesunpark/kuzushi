# Observation Framework

## Purpose

This document defines the core observation domains used in Kuzushi and connects them to observable learning behaviors, recurring patterns, and possible future coding categories. The framework is intended to support longitudinal pattern tracking, synthesis, exploratory visualization, and future trend analysis.

The categories described here are not formal assessment categories. They are used to organize observations over time while preserving the exploratory and naturalistic nature of the project.

## Core Observation Domains

The current observation framework organizes recurring learning behaviors into several broad domains.

| Domain                                      | What is Observed                                                                                        | Example Signals                                                                       |
| ------------------------------------------- | ------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| Cognitive stamina                           | Ability to sustain task engagement over time                                                            | worksheet output, return after breaks, longer task sequences                          |
| Strategy adaptation                         | Changes in approach under constraint                                                                    | familiar-pattern-first approaches, rotation, scaffold fading                          |
| Co-regulation and recovery                  | Emotional recovery after challenge or uncertainty                                                       | openness toward reframing, self-directed regulation, return to task                   |
| Motivation and visible progress             | Impact of rewards, acknowledgment, novelty, and visible completion                                      | response to reward value, completed books, stickers, progress markers                 |
| Spatial reasoning and pattern recognition   | Ability to detect, compare, and manipulate visual or numerical structures                               | part-whole spatial structure, geometric sequences, shape rotation                     |
| Functional arithmetic and early abstraction | Movement from concrete quantities toward symbolic or conceptual reasoning                               | assigning numerical values to object quantities, number bonds, practical arithmetic   |
| Autonomy and independence                   | Ability to initiate, sustain, or complete tasks with reduced direct support                             | task initiative, confidence, creative construction, independent completion            |
| Familiarity, constraint, and flexibility    | Ability to use familiar entry points while adjusting to new task demands                                | adjusting puzzle pieces, adapting known strategies, hybrid problem-solving approaches |
| Cross-context signals                       | Possible recurring problem-solving patterns across academic, game-based, spatial, and embodied contexts | familiarity as an entry point in both academic and embodied contexts                  |

## Academic Learning Behaviors

This domain tracks how learners engage with structured academic tasks, including early math, worksheet completion, numeral writing, functional arithmetic, and emerging abstraction. Academic learning behaviors are observed through task engagement, conceptual use, and the learner’s ability to apply or revisit previously introduced concepts.

Observable signals may include:

* task initiation
* task persistence
* worksheet output
* accuracy and self-correction
* conceptual retention
* numeral writing
* number bonds
* functional arithmetic
* part-whole reasoning
* concrete-to-symbolic reasoning
* application of math in play-based contexts

This domain helps track how learners move from completing isolated tasks toward using mathematical ideas across different formats, including worksheets, games, spatial puzzles, and play-based scenarios.

## Regulation and Co-Regulation

This domain focuses on how learners respond to challenge, uncertainty, comparison, frustration, or loss. Regulation is observed not only through emotional difficulty, but also through recovery, re-engagement, and the supports that help learners return to task.

Self-regulation refers to learner-initiated resets, pauses, returns, or adjustments. Co-regulation refers to support from a trusted adult through reassurance, reframing, proximity, modeling, or calm continuation. Both forms of regulation are important for understanding how learners persist under constraint and recover after disruption.

Observable signals may include:

* response to difficulty
* emotional recovery
* comparison dynamics
* disappointment after loss
* brief disengagement or withdrawal
* self-directed reset
* adult reassurance or reframing
* return to task after disruption
* recovery time

Future observations may examine whether learner-specific regulation patterns appear consistently across academic, spatial, game-based, and embodied contexts.

## Motivation and Reward Systems

This domain tracks how motivation appears through task choice, persistence, re-engagement, response to novelty, and willingness to continue after challenge. Rewards are not treated only as incentives. In Kuzushi, reward systems are also observed as part of the learning environment, shaping anticipation, pacing, choice, completion, and recognition.

Motivational signals may include:

* visible progress markers
* stickers or reward images
* completed books
* novelty
* reward value
* competitive games
* acknowledgment from trusted adults
* learner choice
* willingness to continue after challenge
* preference for familiar or novel task formats

This domain helps track how different learners respond to reward structures over time. Some learners may respond strongly to visible completion and acknowledgment, while others may respond more strongly to novelty, competition, reward value, or choice.

## Spatial Reasoning and Problem-Solving

This domain tracks how learners approach spatial puzzles and other tasks involving visual organization, shape manipulation, pattern recognition, geometric discrimination, and part-whole reasoning. Spatial tasks are especially useful for observing problem-solving under different levels of constraint.

Different constraint levels may reveal different forms of problem-solving behavior. Less constrained tasks may reveal exploration, aesthetic preference, visual organization, curiosity, or intrinsic interest. Higher-constraint tasks may reveal strategy adaptation, frustration recovery, scaffold use, and flexible adjustment to new task demands.

Observable signals may include:

* spatial-fitting puzzles
* shape rotation
* part-whole structure
* compact placement
* visual pattern recognition
* geometric equivalence
* familiar-shape-first approaches
* close-packing strategies
* trial-and-error adjustment
* scaffold fading
* response to changing constraint levels

This domain distinguishes between an entry strategy and adaptation. An entry strategy refers to how a learner begins a task, such as selecting familiar pieces or completing familiar patterns first. Adaptation refers to how the learner adjusts when the task changes, when the initial approach stops working, or when new constraints become visible.

## Cross-Context Observation Thread

Cross-context observations are used cautiously to examine whether similar problem-solving patterns appear across academic, spatial, game-based, and embodied contexts. Periodic youth Jiu Jitsu observations provide a secondary context for observing strategy selection, regulation, persistence, and adaptation under physical, cognitive, and social constraints.

These observations are treated as exploratory signals, not as evidence of transfer or causation. Cross-context observations are included only when they are relevant to possible recurring patterns in problem-solving under constraint.

Examples of cross-context signals may include:

* familiar entry points under pressure
* strategy selection under constraint
* recovery after difficulty
* persistence during challenge
* adaptation when a familiar approach stops working
* similar behavioral patterns across academic, spatial, game-based, and embodied contexts

## Example Observation Categories

The following are example observation categories derived from the core domains. These categories may be used to develop a future data schema or codebook.

| Category           | Example Observation                                                   |
| ------------------ | --------------------------------------------------------------------- |
| Time marker        | session number, week, cycle phase                                     |
| Task type          | worksheet, spatial puzzle, chess, pretend shopping, Jiu Jitsu         |
| Support type       | teacher modeling, reassurance, scaffolding, reframing                 |
| Regulation event   | frustration, comparison, loss, overwhelm, recovery                    |
| Strategy observed  | familiar-pattern-first, rotation, trial-and-error, close-packing      |
| Motivation signal  | reward seeking, novelty seeking, visible completion, competitive play |
| Constraint level   | low, moderate, high, scaffolded                                       |
| Outcome marker     | completion, re-engagement, strategy shift, independent solution       |
| Cross-context flag | academic, spatial, game-based, embodied, possible recurring pattern   |

## Future Coding Considerations

The observation framework defined in this document may later support a structured data schema or codebook. Categories will remain exploratory, and numerical ratings will be used to support pattern tracking rather than learner evaluation. These ratings are not formal assessments or validated measures.

Future coding will prioritize pattern tracking over learner evaluation, prediction, diagnosis, or ranking. Qualitative notes will remain the primary source of interpretation, and cross-context flags will remain secondary and handled with caution to avoid claims of transfer or causation. Coding categories may change as new patterns emerge.

Possible future fields include:

* learner ID
* date range or session number
* task type
* cognitive domain
* support type
* regulation event
* strategy observed
* motivation signal
* constraint level
* outcome marker
* cross-context flag
* notes
